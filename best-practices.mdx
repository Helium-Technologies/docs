## Store keys securely

Never commit API keys to source control or expose them in client-side code. Use environment variables or a secrets manager.

```bash
export NINJACHAT_API_KEY="nj_sk_YOUR_API_KEY"
```

## Handle rate limits gracefully

Check the `Retry-After` header on 429 responses and implement exponential backoff. See [Pricing](/pricing) for rate limit details.

## Monitor your credit balance

Every response includes a `cost` field showing what the request cost. Keep an eye on your balance from [Settings > API](https://www.ninjachat.ai/settings?tab=api) and add credits before running out — requests with insufficient credits return a `402` error.

## Keep your subscription active

API requests require both an active $5/month subscription and sufficient credits. If your subscription lapses, requests return a `403` error.

## Use streaming for chat

For user-facing applications, set `stream: true` in chat requests to display responses as they generate. This provides a better user experience than waiting for the full response.

```json
{
  "messages": [{"role": "user", "content": "Hello!"}],
  "stream": true
}
```

## Write specific image prompts

Include style, lighting, detail level, and composition keywords for better image generation results:

```json
{
  "prompt": "A cyberpunk cityscape at night with neon signs and rain, photorealistic, 8k detail"
}
```

## Rotate keys periodically

Create new keys and revoke old ones on a regular schedule. If a key is compromised, [revoke it immediately](https://www.ninjachat.ai/settings?tab=api).
