The video generation API supports 5 models through a single endpoint. Submit a job, poll for completion, and receive a CDN-hosted MP4 URL.

## Available models

| Model | ID | Description | Duration | Speed | Best for |
|---|---|---|---|---|---|
| Seedance 2 | `seedance-2` | Default model. Uncensored, no content filters. Longest duration support. | 5–15s | ~1-2 min | Unrestricted content |
| Kling Video | `kling-video` | Kuaishou's model. Excellent motion quality and scene coherence. | 5–10s | ~3-4 min | Motion quality |
| Google Veo 2 | `google-veo-2` | Google's fast cinematic model. High quality with quick generation. | 5–8s | ~40s | Fast, high-quality |
| Google Veo 3.1 | `veo-3.1` | Google DeepMind's premium model. Cinematic quality, photorealistic detail. | 4, 6, or 8s | ~3-5 min | Premium cinematic |
| Veo 3.1 Fast | `veo-3.1-fast` | Speed-optimized Veo 3.1. Same architecture, 2-3x faster generation. | 4, 6, or 8s | ~1-2 min | Speed + quality |

All models cost **$3.00 per video**. Credits are refunded automatically if generation fails.

## How it works

Video generation is asynchronous. The flow is:

1. **Submit** — `POST /api/v1/video` with your prompt and model. Returns a `request_id` immediately.
2. **Poll** — `GET /api/v1/video/status?request_id=...` until `status` is `completed` or `failed`.
3. **Use** — The completed response contains a `video_url` pointing to a CDN-hosted MP4.

<Note>
Polling the status endpoint is free and does not count toward your rate limit or credits.
</Note>

## Text-to-video

Generate a video from a text prompt:

<CodeGroup>

```bash cURL
curl -X POST https://ninjachat.ai/api/v1/video \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer nj_sk_YOUR_API_KEY" \
  -d '{
    "prompt": "A drone shot soaring over a misty mountain range at golden hour",
    "model": "google-veo-2",
    "duration": 5,
    "aspect_ratio": "16:9"
  }'
```

```python Python
import requests

response = requests.post(
    "https://ninjachat.ai/api/v1/video",
    headers={
        "Content-Type": "application/json",
        "Authorization": "Bearer nj_sk_YOUR_API_KEY",
    },
    json={
        "prompt": "A drone shot soaring over a misty mountain range at golden hour",
        "model": "google-veo-2",
        "duration": 5,
        "aspect_ratio": "16:9",
    },
)

data = response.json()
print(data["request_id"])
```

```javascript JavaScript
const response = await fetch("https://ninjachat.ai/api/v1/video", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer nj_sk_YOUR_API_KEY",
  },
  body: JSON.stringify({
    prompt: "A drone shot soaring over a misty mountain range at golden hour",
    model: "google-veo-2",
    duration: 5,
    aspect_ratio: "16:9",
  }),
});

const data = await response.json();
console.log(data.request_id);
```

</CodeGroup>

## Polling for completion

After submitting, poll the status endpoint until the video is ready:

<CodeGroup>

```python Python
import time
import requests

# After submitting a video job...
request_id = data["request_id"]

while True:
    status = requests.get(
        f"https://ninjachat.ai/api/v1/video/status?request_id={request_id}",
        headers={"Authorization": "Bearer nj_sk_YOUR_API_KEY"},
    ).json()

    if status["status"] == "completed":
        print(f"Video ready: {status['result']['video_url']}")
        break
    elif status["status"] == "failed":
        print(f"Failed: {status.get('error')}")
        break

    print(f"Processing... {status.get('progress', 0)}%")
    time.sleep(10)
```

```javascript JavaScript
// After submitting a video job...
const requestId = data.request_id;

while (true) {
  const statusRes = await fetch(
    `https://ninjachat.ai/api/v1/video/status?request_id=${encodeURIComponent(requestId)}`,
    { headers: { "Authorization": "Bearer nj_sk_YOUR_API_KEY" } }
  );
  const status = await statusRes.json();

  if (status.status === "completed") {
    console.log("Video ready:", status.result.video_url);
    break;
  } else if (status.status === "failed") {
    console.error("Failed:", status.error);
    break;
  }

  console.log(`Processing... ${status.progress || 0}%`);
  await new Promise(r => setTimeout(r, 10000));
}
```

</CodeGroup>

## Error handling

If generation fails, you are **not charged**. Credits are automatically refunded. The status response will show:

```json
{
  "request_id": "gemini-veo:operations/...",
  "status": "failed",
  "error": "Video generation failed"
}
```

Common failure reasons:
- Content blocked by safety filters (Veo and Kling models — Seedance 2 has no filters)
- Upstream service temporarily unavailable
- Invalid image URL for image-to-video
